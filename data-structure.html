📜 Свиток структуры проклятых данных

🏢 Миссия (Mission)

Атрибуты:

  • missionId: String - уникальный идентификатор миссии
  • date: LocalDate - дата проведения
  • location: String - место проведения
  • outcome: Outcome (SUCCESS/FAILURE) - результат
  • damageCost: int - стоимость ущерба
  • curse: Curse - проклятие-цель (композиция)
  • sorcerers: List - участвующие маги (агрегация)
  • techniques: List - использованные техники (композиция)

👹 Проклятие (Curse)

Атрибуты:

  • name: String - имя проклятия
  • threatLevel: ThreatLevel (LOW/MEDIUM/HIGH/SPECIAL_GRADE) - уровень угрозы

← Связано с миссией (одна миссия - одно проклятие)

⚔️ Маг (Sorcerer)

Атрибуты:

  • name: String - имя мага
  • rank: Rank (GRADE_4/GRADE_3/GRADE_2/GRADE_1/SPECIAL_GRADE) - ранг

← Участвует в миссии (много магов в одной миссии)

🔮 Техника (Technique)

Атрибуты:

  • name: String - название техники
  • type: TechniqueType (INNATE/CURSED_TOOL/SHIKIGAMI/BARRIER) - тип
  • owner: String - владелец (имя мага)
  • damage: int - нанесенный урон

← Принадлежит магу, используется в миссии

📊 Связи между сущностями:

Связь Тип Описание
Mission → Curse Композиция У каждой миссии есть ровно одно целевое проклятие
Mission → Sorcerer Агрегация В миссии участвует от 1 до N магов
Mission → Technique Композиция В миссии используется от 0 до N техник
Technique → Sorcerer Ассоциация Техника принадлежит конкретному магу (по имени)
